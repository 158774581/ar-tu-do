1 # Autonomous Racing Software Stack and Simulation Enviroment
2 
3 [![Build Status](https://travis-ci.com/Autonomous-Racing-PG/ros.package.svg?branch=master)](https://travis-ci.com/Autonomous-Racing-PG/ros.package)
4 
5 This repository contains software for 1/10th scale autonomous race cars to compete in the [F1/10 competition](http://f1tenth.org/). It is developed by the Autonomous Racing Project Group of [TU Dortmund](https://ls12-www.cs.tu-dortmund.de/daes/).
6 
7 ![](doc/racing_example.gif "Racing with a wallfollowing algorithm")
8 
9 ## Features
10 
11 We provide several LIDAR based driving algorithms:
12 
13 - Fast and efficient wallfollowing based on fitting circles into the LIDAR scan
14 - [ROS navigation stack](http://wiki.ros.org/navigation) based implementation that uses SLAM, a precalculated map and path planning
15 - Deep Reinforcement Learning ([Q-Learning](https://en.wikipedia.org/wiki/Q-learning))
16 - Neural Networks with evolutionary training
17 
18 Our software works on physical hardware and in a simulated environment using [Gazebo](http://gazebosim.org/).
19 Further features are:
20 
21 - Automatic emergency braking
22 - Dead Man's Switch
23 - Teleoperation via keyboard, Xbox and Playstation controller
24 - Speedometer and Lap Timer

27 ## Installation
28 
29 You need to install the [Robot Operating System (ROS)](https://www.ros.org/) to use our software. We target [ROS Kinetic](http://wiki.ros.org/kinetic/Installation) on Ubuntu 16.04, but [ROS Melodic](http://wiki.ros.org/melodic/Installation) seems to work as well.
30 
31 Note for VM users: Gazebo 7.0.0, which is installed with ROS Kinetic by default, [does not work](https://bitbucket.org/osrf/gazebo/issues/1837/vmware-rendering-z-ordering-appears-random) on a virtual machine. To solve this, Gazebo has to be updated to at least 7.4.0 [as explained here](http://gazebosim.org/tutorials?cat=install&tut=install_ubuntu&ver=7.0#Alternativeinstallation:step-by-step).
32 
33 Install dependencies:
34 
35 ```bash
36 sudo pip uninstall pip && sudo apt install python-pip
37 sudo apt install libsdl2-dev clang-format python-pyqtgraph
38 pip install torch autopep8 cython
39 
40 # RangeLibc
41 git clone http://github.com/kctess5/range_libc
42 cd range_libc/pywrapper
43 # Either:
44 ./compile.sh # on VM
45 # Or:
46 ./compile_with_cuda.sh # on car - compiles GPU ray casting methods
47 ```
48 
49 Clone the repository:
50 
51 ```bash
52 git clone --recurse-submodules https://github.com/Autonomous-Racing-PG/ros.package
53 ```
54 
55 Install missing ROS dependencies:
56 
57 ```bash
58 cd ros.package/ros_ws
59 rosdep install -y --from-paths src --ignore-src --rosdistro ${ROS_DISTRO}
60 ```

63 ## Usage
64 
65 Compile while inside the `ros_ws`directory:
66 
67 ```bash
68 catkin_make
69 ```
70 
71 Set up enviroment variables for your shell:
72 
73 ```bash
74 source devel/setup.bash # (or setup.zsh, depending on your shell)
75 ```
76 
77 Use a launch file to start ROS and Gazebo:
78 
79 ```bash
80 roslaunch launch/car.launch # (Physical car, Wallfollowing)
81 roslaunch launch/gazebo.launch # (Simulation, Wallfollowing)
82 roslaunch launch/navigation_stack.launch # (Simulation, SLAM & ROS navigation)
83 roslaunch launch/qlearning.launch # (Simulation, Train the Q-Learning model)
84 roslaunch launch/evolutionary.launch # (Simulation, Train the evolutionary neural network)
85 ```
86 
87 ### Launch file arguments
88 
89 These arguments can be passed to the launch files above. For example, to use the `gazebo.launch` file without emergency stop and with car highlighting, run:
90 ```
91 roslaunch launch/gazebo.launch emergency_stop:=false car_highlighting:=true
92 ```
93 The arguments can be changed permanently by editing the launch files.

237 ## Hardware
238 
239 Our car is based on a 1/10th scale RC car ([Traxxas Ford Fiesta](https://traxxas.com/products/models/electric/ford-fiesta-st-rally)) with these additions:
240 
241 - CPU/GPU board ([NVIDIA Jetson](https://www.nvidia.com/object/jetson-tk1-embedded-dev-kit.html))
242 - motor controller ([FOCBOX](https://www.enertionboards.com/FOCBOX-foc-motor-speed-controller.html))
243 - LIDAR scanner ([Hokuyo UST-10LX](https://www.hokuyo-usa.com/products/scanning-laser-rangefinders/ust-10lx))
244 - an inertial measurement unit ([Invensense MPU-9250](https://www.invensense.com/products/motion-tracking/9-axis/mpu-9250/))
245 - optional: brushless DC motor (replaces the standard brushed motor)
246 - optional: stereo camera ([ZED](https://www.stereolabs.com/zed/))
